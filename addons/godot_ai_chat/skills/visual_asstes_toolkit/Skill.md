# Visual Arts Toolkit

## 概览
根据任务描述，产出指定的美术资产。

## 工作流

1. **分析需求**：根据任务描述确定产出类型（材质/着色器/静态贴图等）
2. **盘点资产**：检查指定工作区中已有的美术资产。
3. **创建文件**：根据任务描述和实际情况，进行相应的美术资产创建。
   - 缺贴图 → 生成静态图像
   - 缺材质 → 创建材质资源
   - 缺着色器 → 编写着色器代码
4. **确认结果**：检查产出的美术资产是否正确创建
5. **报告**：使用 `report_task_result` 进行报告

## 注意事项

- 如果遇到同名文件存在，重新起一个不冲突的文件名。
- PBR材质贴图需遵循标准 PBR 命名（`_albedo`, `_normal` 等），否则将无法生成正确的PBR材质。
- `create_resource` 的 `resource_type` 必须指定具体的子类，不能使用基类 `Resource`
- 使用 `search_godot_api` 确定材质资源的 `resource_type`
- 使用 `search_shader_library` 和 `web_fetch_content` 抓取Godot社区用户分享的各类着色器代码，能帮助你写出更好的着色器实现
