# 如何创建自定义技能 (How to create custom Skills)

自定义技能（Custom Skills）允许您利用特定于您的组织或个人工作方式的专业知识和工作流程来增强 Claude。本文将说明如何创建、构建和测试您自己的技能。

技能可以像几行指令一样简单，也可以像包含可执行代码的多文件包一样复杂。

**最好的技能通常具备以下特点：**
*   解决一个特定的、可重复的任务
*   拥有 Claude 可以遵循的清晰指令
*   在有帮助时包含示例
*   定义了何时应该使用它们
*   专注于一个工作流程，而不是试图做所有事情

---

### 创建 Skill.md 文件

每个技能都包含一个目录（文件夹），该目录中至少包含一个 `Skill.md` 文件，这是技能的核心。

此文件必须以 **YAML frontmatter**（元数据头）开头，以包含名称 (`name`) 和描述 (`description`) 字段，这是必需的元数据。它还可以包含其他元数据、给 Claude 的指令、参考文件、可执行脚本或工具。

#### 必需的元数据字段

*   **name (名称)**：您的技能的人类友好名称（最多 64 个字符）。
    *   *示例：* `Brand Guidelines`
*   **description (描述)**：关于该技能做什么以及何时使用它的清晰描述。这至关重要——Claude 使用它来决定何时调用您的技能（最多 200 个字符）。
    *   *示例：* `Apply Acme Corp brand guidelines to presentations and documents, including official colors, fonts, and logo usage.`

#### 可选的元数据字段

*   **dependencies (依赖项)**：您的技能所需的软件包。
    *   *示例：* `python>=3.8, pandas>=1.5.0`

> `Skill.md` 文件中的元数据充当“渐进式披露”系统的第一层，提供足以让 Claude 知道何时应使用该技能的信息，而无需加载所有内容。

#### Markdown 正文
Markdown 正文是元数据之后的第二层详细信息，因此 Claude 将在读取元数据后根据需要访问此内容。根据您的任务，Claude 可以访问 `Skill.md` 文件并使用该技能。

---

### 添加资源

如果您有太多信息无法添加到单个 `Skill.md` 文件中（例如，仅适用于特定场景的部分），您可以通过在技能目录中添加文件来添加更多内容。

例如，向您的技能目录添加一个包含补充和参考信息的 `REFERENCE.md` 文件。在 `Skill.md` 中引用它将有助于 Claude 决定在执行技能时是否需要访问该资源。

### 添加脚本

对于更高级的技能，请将可执行代码文件附加到 `Skill.md`，允许 Claude 运行代码。例如，我们的文档技能使用以下编程语言和包：
*   Python (pandas, numpy, matplotlib)
*   JavaScript/Node.js
*   用于帮助文件编辑的包
*   可视化工具

---

### 打包您的技能

一旦您的技能文件夹完成：
1.  确保文件夹名称与您的技能名称匹配。
2.  创建该文件夹的 **ZIP 文件**。
3.  ZIP 包应以技能文件夹作为其根目录（而不是子文件夹）。

### 上传到 Claude
1.  导航至 **Settings (设置) > Capabilities (功能)**。
2.  确保 **Code execution and file creation (代码执行和文件创建)** 已启用。
3.  滚动到 **Skills (技能)** 部分。
4.  点击 **"Upload skill" (上传技能)** 并上传您的 ZIP 文件。

### 最佳实践

*   **保持专注 (Keep it focused)**：为不同的工作流程创建单独的技能。多个专注的技能比一个大型技能组合效果更好。
*   **编写清晰的描述 (Write clear descriptions)**：Claude 使用描述来决定何时调用您的技能。具体说明它何时适用。
*   **从简单开始 (Start simple)**：在添加复杂的脚本之前，先从 Markdown 中的基本指令开始。您以后随时可以扩展该技能。
*   **使用示例 (Use examples)**：在您的 `Skill.md` 文件中包含示例输入和输出，以帮助 Claude 理解成功的样子。
*   **增量测试 (Test incrementally)**：在每次重大更改后进行测试，而不是一次性构建复杂的技能。
*   **技能可以相互构建**：虽然技能不能显式引用其他技能，但 Claude 可以自动一起使用多个技能。这种可组合性是技能功能最强大的部分之一。

### 安全注意事项

*   在向 `Skill.md` 文件添加脚本时请务必谨慎。
*   **不要硬编码敏感信息**（如 API 密钥、密码）。
*   在启用下载的任何技能之前，请对其进行审查。
*   使用适当的 MCP 连接进行外部服务访问。
