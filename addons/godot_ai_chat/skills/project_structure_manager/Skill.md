## Project Structure Manager

### 概览
本技能旨在指导如何整理项目中散乱的文件与目录，为游戏项目管理打下基础。

### 可用工具
- `read_file` — 读取文件内容
- `rename_file` — 重命名文件
- `move_file` — 移动文件
- `manage_folder` — 管理文件夹
- `manage_todo_list` — 管理待办事项
- `open_file` — 在编辑器中打开场景
- `get_node_properties` — 获取节点属性

### 工作流程

#### 1. 全局扫描
扫描项目根目录下的文件与文件夹结构，了解整体布局与资源分布情况。

#### 2. 制定计划
根据扫描结果，结合以下整理概念，制定具体的整理计划。使用 `manage_todo_list` 将计划拆解为可追踪的待办事项。

**整理概念：**
- **模块归拢** — 将属于同一功能实体的文件集中到独立的模块文件夹中，遵循一个文件夹 = 一个功能实体的原则
- **命名规范化** — 将文件名统一为 `snake_case`
- **共享资源提取** — 被多模块引用的通用资源移至 `res://shared/` 下对应分类
- **暂存机制** — 无法确定归属的文件先移入 `res://temp/`，并在其中按文件类型再建立不同的子文件夹并分类存放

#### 3. 执行计划
依次完成待办事项中的各项整理任务，每完成一项使用 `manage_todo_list` 标记完成。等所有任务执行完毕后，使用 `report_task_result` 工具进行汇报。
